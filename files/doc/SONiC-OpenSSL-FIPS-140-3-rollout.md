## Overview
It is for the security requirement, the FIPS 140-3 should be enforced for the devices in the datacenters.

## Requirements
- Deploy SONIC version for FedRAMP Fleet to be FIPS compliant, see Feature [#24944353](https://msazure.visualstudio.com/One/_workitems/edit/24944353)

## Scopes
1. The FIPS 140-3 is only availabel on SONiC OS Version 11 or above.
2. FIPS is supported on 202305+ branches.
3. FIPS is only supported on architectures AMD64, ARM64, but not supported on archetecture ARMHF currently, for instance, Nokia 7215.

## How to enable FIPS
There is a FIPS default option to enable FIPS by default when building the SONiC images. If the The FIPS option is turned on, then the FIPS will be enforced when installing the image. Currently, the option is not set, so the FIPS is in disabled status by default. For the internal release 202305+, We can turn on the option in the image to support the FIPS deployment. See Pull Request [#8808723: [202305] Change the FIPS default option in image](https://msazure.visualstudio.com/DefaultCollection/One/_git/Networking-acs-buildimage/pullrequest/8808723).

Since the config option is in the installation script in the image, it can support all the scenarios easily, such as upgrading for any old images, converting from EOS images, installation from any bootloaders, aboot, grub and uboot.

## FIPS Design State
The image FIPS default option should be the same as the design state in Network Graph Metadata, used to generate golden config generated by NDM. See the liquid config template in Pull Request [#8342173: Add the liquid template for SONiC FIPS](https://msazure.visualstudio.com/DefaultCollection/One/_git/Networking-Metadata/pullrequest/8342173).

If the desgin state changed, a device FIPS config is not the same with the image default value, then it is required to reboot the device to align the design state after upgrading the the new image.

## FIPS Design State change improvement (May be not need to support)
To make up the disadvantage of the FIPS disabled scenario, we can add do some change in the reboot (including fast/warm reboot).

For the image 202305+, see Pull Request [#2998: Support to upgrade the FIPS config during the reboot](https://github.com/sonic-net/sonic-utilities/pull/2998).

For the image 202305 below, similar change on the patch, see Pull Request [#3680 Support sonic fips upgrade](https://dev.azure.com/mssonic/internal/_git/sonic-metadata/pullrequest/3680).

## Test cases
### Test case #1 – Test to install a FIPS enabled image, cover aboot, grub, uboot
1. Setup the test environment to the FIPS disabled.
1. Install the image
1. Reboot
1. Verify the FIPS enabled

### Test case #1 – Test to design state change
1. Setup the test environment to the FIPS Enabled.
1. Enable the FIPS in golden config db and apply the change to config db.
1. Reboot
1. Verify the FIPS disabled
